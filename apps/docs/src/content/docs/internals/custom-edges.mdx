NgDiagram provides two approaches for edge customization:

- [**Default edge customization**](#customizing-default-edges) - For simpler styling needs like custom colors, arrowheads, and using built-in routing algorithms, you can customize the default edges with CSS variables and edge data parameters.
- [**Custom edge components**](#implementing-custom-edges) - For more complex requirements such as custom routing logic, animations, or unique visual representations, you can create fully custom edge components by implementing `NgDiagramEdgeTemplate`.

## Customizing Default Edges

NgDiagram's default edges offer several customization options:

- Custom arrowheads (defined as SVG markers)
- CSS variable styling
- Built-in routing algorithms (straight, orthogonal, bezier)

## Custom Routing

To use custom routing set the `pathAndPoints` property of `NgDiagramBaseEdgeComponent`. The `path` property defines the SVG path data for the edge, while `points` defines the control points for the edge's routing. The following example demonstrates how to create a sinusoidal edge between nodes:
